Bài trước mình đã giới thiệu và hướng dẫn cách cài đặt Laravel. Bài hôm nay mình xin nói sơ lượt về cấu trúc thư mục trong Laravel 5 cụ thể ở đây là Laravel phiên bản 5.4.

Cấu trúc thư mục trong Laravel 5:

- app: Chứa models, controllers, middlewares, request, ... đây là thư mục code chính mà bạn sẽ thao tác nhiều sau này.
- bootstrap: Chứa một số file của bootstrap framework và một số file cấu hình nạp tự động.
- config: Chứa các file cấu hình hệ thống như database, mail, filesystems, ...
- database: migrations: nơi bạn sẽ khởi tạo database, seeds: nơi sẽ khởi tạo dữ liệu insert mẫu vào database.
- public: Thư mục public cũng dùng cho việc lưu trữ những file như css, javascripts, file images và những file khác. Nó cũng chứa file khởi động index.php.
- resources: Chứa assets, lang, views (chứa file cấu trúc html)
- routes: Chứa các file api.php, channels.php, console.php, web.php. Ở đay ta chủ yếu quan tâm tới file web.php nó lưu trứ toàn bộ những route để thông báo với Laravel cách như thế nào để kết nối khi có yêu cầu, theo hàm khai sẵn, controller và action.
- storage: Để lưu trữ file tạm thời cho những dịch vụ Laravel khác nhau như session, cache, biên dịch template views.
- tests: Chứa các test tự động của bạn.
- vendor: Nơi này chứa toàn bộ code của bên thứ ba. Chứa plugin chúng ta cài thêm cho ứng dụng chúng ta.Chứa cả các composer package sử dụng bởi ứng dụng.

Trong bài này mình đã giới thiệu sơ lượt cấu trúc thư mục trong Laravel 5. Trong series bài viết tiếp theo mình sẽ hướng dẫn các bạn về Route trong Laravel 5. Mời các bạn đón đọc. Bài viết trên là những hiểu biết của cá nhân mình nên không tránh phải sai sót, rất mong sự đóng góp ý kiến của các bạn để bài viết trở nên hữu ích hơn.
